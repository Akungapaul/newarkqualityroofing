  <name>Task 1: Create Zod schemas and TypeScript types</name>
  <files>
    src/lib/schemas.ts
    src/lib/types.ts
  </files>
  <action>
    **1. Create `src/lib/schemas.ts`:**

    Define Zod schemas for all page types:

    - `ServiceCategorySchema`: z.enum with 8 categories: 'repair-maintenance', 'residential-roof-types', 'commercial-roof-types', 'components-specialty', 'energy-solar', 'commercial-services', 'design-consultation', 'replacement-sub-pages'

    - `ServiceSchema`: z.object with fields:
      - id: z.string() (kebab-case unique identifier)
      - name: z.string() (display name)
      - slug: z.string().regex(/^[a-z0-9-]+$/) (URL slug)
      - category: ServiceCategorySchema
      - parentId: z.string().nullable() (for parent-child hierarchy, e.g. replacement sub-pages)
      - isResidential: z.boolean()
      - isCommercial: z.boolean()
      - shortDescription: z.string() (1-2 sentence summary for cards/lists)
      - metaTitle: z.string().max(60)
      - metaDescription: z.string().max(160)

    - `CitySchema`: z.object with fields:
      - id: z.string()
      - name: z.string()
      - slug: z.string().regex(/^[a-z0-9-]+$/) (the raw city slug like "montclair", NOT the full page slug)
      - state: z.literal('NJ')
      - county: z.literal('Essex')
      - isHQ: z.boolean().default(false)
      - zipCodes: z.array(z.string())
      - adjacentCityIds: z.array(z.string()) (for nearby cities block)

    - `ComboSchema`: z.object with fields:
      - slug: z.string().regex(/^[a-z0-9-]+$/)
      - serviceId: z.string()
      - cityId: z.string()
      - metaTitle: z.string().max(70) (allow slightly longer for "{service} in {city}, NJ")
      - metaDescription: z.string().max(160)

    - `ComparisonSchema`: z.object with fields:
      - id: z.string()
      - name: z.string() (e.g. "Asphalt Shingles vs Metal Roofing")
      - slug: z.string().regex(/^[a-z0-9-]+$/)
      - category: z.enum(['material-vs-material', 'service-vs-service', 'decision-helper'])
      - itemA: z.string().optional() (first item being compared)
      - itemB: z.string().optional() (second item being compared)
      - metaTitle: z.string().max(60)
      - metaDescription: z.string().max(160)

    - `CorePageSchema`: z.object with fields:
      - id: z.string()
      - name: z.string()
      - slug: z.string() (can be empty string for homepage handled by app/page.tsx)
      - metaTitle: z.string().max(60)
      - metaDescription: z.string().max(160)

    - `PageType`: z.enum(['service', 'city', 'combo', 'comparison', 'article', 'core'])

    - `SlugEntrySchema`: z.object with fields:
      - slug: z.string()
      - type: PageType
      - serviceId: z.string().optional()
      - cityId: z.string().optional()
      - comparisonId: z.string().optional()
      - corePageId: z.string().optional()

    **2. Create `src/lib/types.ts`:**

    Export inferred TypeScript types from all Zod schemas using `z.infer<typeof Schema>`. This file is the single import point for types throughout the app:
    ```typescript
    export type Service = z.infer<typeof ServiceSchema>;
    export type City = z.infer<typeof CitySchema>;
    // etc.
    ```

    Also re-export schemas from schemas.ts for convenience.
  </action>
  <verify>
    Run `npx tsc --noEmit` -- zero type errors.
    Confirm schemas.ts exports all schemas and types.ts exports all inferred types.
  </verify>
  <done>
    All Zod schemas defined with proper constraints. TypeScript types inferred from schemas. Single source of truth for data shapes.
  </done>
