    **0. Create `src/data/nav-data.ts` (CRITICAL -- prevents Zod client bundling):**

    The Header is a "use client" component for dropdown/menu interactivity. If it directly imports `src/data/services.ts` or `src/data/cities.ts`, Zod (~50KB) gets bundled into every page's client JS because those files call `z.array(...).parse()` at module level.

    Instead, create a `src/data/nav-data.ts` file that:
    1. Imports from `services.ts` and `cities.ts` (which triggers Zod validation at build time on the server)
    2. Pre-computes the navigation menu structure into plain objects/arrays with NO Zod imports
    3. Exports simple, serializable data structures

    ```typescript
    // src/data/nav-data.ts
    // This file is imported by client components (Header, MobileMenu).
    // It must NOT import Zod or any module that calls .parse() at module level.
    // Instead, it exports pre-computed plain data.

    // Type definitions for nav menu items (plain interfaces, no Zod)
    export interface NavServiceItem {
      name: string;
      slug: string;
    }

    export interface NavServiceGroup {
      category: string;
      categoryLabel: string;
      services: NavServiceItem[];
    }

    export interface NavCityItem {
      name: string;
      slug: string; // Full page slug like "roofing-in-newark-nj"
    }

    // Pre-computed data -- these are plain object literals, NOT runtime-computed
    // from Zod-validated sources. Generate these values by reading from the
    // services.ts and cities.ts data at CODE AUTHORING time (when this file is written),
    // not at runtime.
    //
    // ALTERNATIVE APPROACH (recommended): Make this a server-only module that computes
    // the nav data, then pass the data as props from a server component wrapper.
    // See Header implementation below for the server/client split pattern.

    export const serviceMenuGroups: NavServiceGroup[] = [
      // ... generated from services data, grouped by category
    ];

    export const cityMenuItems: NavCityItem[] = [
      // ... generated from cities data with full page slugs
    ];
    ```

    **IMPORTANT -- Two valid approaches, pick the cleaner one:**

    **Approach A (Static nav-data.ts):** Hardcode the navigation arrays in `nav-data.ts` as plain object literals. No imports from services.ts/cities.ts. Downside: data is duplicated, must stay in sync manually.

    **Approach B (Server component wrapper -- PREFERRED):** Keep `nav-data.ts` as a server-only module that DOES import from services.ts/cities.ts and computes the menu structure. Then create a thin server component wrapper in `layout.tsx` that calls nav-data functions and passes the result as serialized props to the client Header component. This way Zod runs only on the server.

    Use Approach B:
    - `nav-data.ts` imports services/cities, groups by category, generates city page slugs
    - `nav-data.ts` exports `getServiceMenuGroups()` and `getCityMenuItems()` functions
    - In `layout.tsx` (server component), call these functions and pass results as props to `<Header serviceGroups={...} cityItems={...} />`
    - Header.tsx ("use client") receives plain serialized props, never imports Zod chain

    **2. Update `src/app/layout.tsx`:**

    Integrate Header and Footer into the root layout. CRITICAL: layout.tsx is a server component. Use it to compute nav data and pass to client Header.

    ```typescript
    // layout.tsx (server component)
    import { getServiceMenuGroups, getCityMenuItems } from '@/data/nav-data';
    import { Header } from '@/components/layout/Header';
    import { Footer } from '@/components/layout/Footer';

    export default function RootLayout({ children }) {
      // Compute nav data on the server (Zod runs here, not in client bundle)
      const serviceGroups = getServiceMenuGroups();
      const cityItems = getCityMenuItems();

      return (
        <html>
          <body>
            <Header serviceGroups={serviceGroups} cityItems={cityItems} />
            <main>{children}</main>
            <Footer />
          </body>
        </html>
      );
    }
    ```
