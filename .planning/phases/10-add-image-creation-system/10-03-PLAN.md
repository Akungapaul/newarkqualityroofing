  <action>
Build the OG image compositing library and the generation script.

**scripts/images/lib/og-compositor.ts** -- OG compositor:
- `createOGImage(options: OGCompositeOptions): Promise<void>` function
- OGCompositeOptions: { backgroundImage: string, title: string, subtitle?: string, outputPath: string }
- Pipeline:
  1. Load background image via sharp
  2. Resize to 1200x630 with fit 'cover', position 'centre'
  3. Create SVG overlay with:
     - Dark forest green gradient from transparent (top) to rgba(15,34,24,0.85) (bottom)
     - Copper (#C17F4E) accent bar: 80px wide, 4px tall, at y=480, x=60, rounded
     - Company name "NEWARK QUALITY ROOFING" in copper, serif font, 18px, letter-spacing 2, at y=510
     - Page title in parchment (#F5F0E8), serif font, 36px bold, at y=560
     - Optional subtitle in parchment at 80% opacity, 20px, at y=595
  4. Composite SVG overlay on photo
  5. Output as WebP quality 85

**Text wrapping for long titles:**
- Calculate approximate text width: character count * 19 (approximate serif char width at 36px)
- If width > 1080px (1200 - 120px margins): split into two lines at nearest word boundary before midpoint
- Line 1 at y=545, line 2 at y=585 (adjust subtitle to y=615)
- Max 2 lines; truncate with ellipsis if still too long

**escapeXml(text: string)** helper: escape &, <, >, ", ' for SVG safety

**scripts/images/og-generate.ts** -- OG generation CLI:
- Parse CLI args: `--dry-run`, `--category <homepage|service|city>`, `--id <page-id>`
- Load dotenv (not needed for API key, but for consistency)

**OG image sources and targets:**
Build list of OG images to generate:
1. Homepage (1): Use hero-main image as background, title "Newark Quality Roofing", subtitle "Expert Roofing Services in Essex County NJ" -> public/images/og/homepage.webp
2. Service pages (65): For each service, use its approved service-hero image as background, title = service.name, subtitle = "Professional Roofing Services" -> public/images/og/{service-slug}.webp
3. City pages (21): For each city, use its approved city-hero image as background, title = "{city.name} Roofing Services", subtitle = "Licensed NJ Roofing Contractor" -> public/images/og/{city-slug}.webp

**Dry-run mode**: List all OG images with source background, title text, output path. Check which backgrounds exist (approved hero images). Warn about missing backgrounds.

**Generation mode**:
- For each OG target, look up the source hero image in public/images/heroes/
- If source image not found (not yet approved), skip with warning
- Call createOGImage() for each
- After completion, add OG entries to manifest via addToManifest() with category='og', appropriate pageType and pageId
- Print summary: X generated, Y skipped (missing backgrounds), paths

**scripts/images/audit.ts** -- Coverage audit:
- Import services, cities from data layer
- Import imageManifest from src/data/image-manifest.ts

**Coverage checks:**
1. **Homepage images**: Check manifest for all required homepage category entries (hero-main, residential-split, commercial-split, why-licensed, why-experience, why-warranty, why-team, pricing-estimate, repair-services, consultation, faq-options, service-map, trust-bar, cta-final, emergency-banner)
2. **Service hero images**: For each of 65 services, check manifest has entry with category='service-hero' and pageId=service.id
3. **City hero images**: For each of 21 cities, check manifest has entry with category='city-hero' and pageId=city.id
4. **Gallery pairs**: Check for at least 6 before/after pairs (gallery-before and gallery-after entries)
5. **Content pool**: Check for at least 4 entries in each content subcategory (crew, materials, consultation, seasonal)
6. **OG images**: For each of homepage + 65 services + 21 cities, check manifest has entry with category='og' and matching pageType/pageId

**Output format:**
- Section headers: "Homepage Images", "Service Heroes", "City Heroes", "Gallery", "Content Pool", "OG Images"
- For each section: X/Y covered (Z%)
- List missing items by name
- Final summary line: "Total coverage: X/Y (Z%)" with pass/fail indicator
- Exit code 0 if all critical coverage met (service heroes, city heroes, OG), exit code 1 if gaps

**Also check for orphaned manifest entries:**
- Entries in manifest that don't correspond to any known page (stale entries from deleted/renamed pages)
- Report as warnings (not errors)

Follow npm script pattern: runs via `npm run images:audit` using tsx.
