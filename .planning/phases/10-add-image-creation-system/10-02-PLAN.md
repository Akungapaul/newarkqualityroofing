  <action>
Create the main generation script and the HTML preview generator.

**scripts/images/generate.ts** -- Main generation CLI:
- Parse CLI args: `--dry-run`, `--category <cat>`, `--id <image-id>`, `--concurrency <n>` (default 3), `--delay <ms>` (default 1000)
- Load dotenv for OPENAI_API_KEY
- Filter prompts by category or id if specified, otherwise use ALL_PROMPTS
- Check for existing staged images and skip already-staged entries (unless `--force` flag)

**Dry-run mode** (`--dry-run`):
- List all images that would be generated in a formatted table: id, category, filename, dimensions
- Show total count, estimated cost (via cost-tracker.estimateCost), and target staging paths
- Do NOT make any API calls
- Exit with summary

**Generation mode** (no --dry-run):
- Validate OPENAI_API_KEY exists in env, exit with clear error if missing
- Create staging directory structure: staging/images/{heroes,gallery,content,homepage}/
- Call generateWithConcurrency() with filtered prompts
- For each successful generation:
  - Write raw PNG to staging/images/{category-subdir}/{filename}.png
  - Log success with timing
- For each failure:
  - Log error with prompt id and error message
  - Continue (don't abort batch)
- After batch completes:
  - Generate report via cost-tracker.generateReport()
  - Write report to scripts/reports/ via cost-tracker.writeReport()
  - Auto-generate preview HTML via the preview module
  - Print summary: X succeeded, Y failed, Z skipped, estimated cost, report path

**scripts/images/preview.ts** -- HTML preview generator:
- Scan staging/images/ for all image files (PNG, WebP, JPG)
- For each image, find the matching PromptDefinition by filename
- Generate a static HTML file at staging/preview.html with:
  - Parchment (#F5F0E8) background, serif fonts (matching project aesthetic)
  - Grid layout: cards with image, id, category badge, dimensions, alt text, and full prompt text
  - Filter buttons at top to show/hide by category
  - Image count and timestamp in header
  - Images displayed at 400px width with lazy loading
  - Relative paths to staged images (so preview.html works when opened locally)
- If called programmatically (exported function), return the HTML string
- If called via CLI, write to staging/preview.html and log the path

<task type="auto">
  <name>Task 2: Approve and status scripts</name>
  <files>
    scripts/images/approve.ts
    scripts/images/status.ts
  </files>
  <action>
Create the approve workflow and status reporting scripts.

**scripts/images/approve.ts** -- Approve staged images:
- Parse CLI args: `--all`, `--category <cat>`, `--id <image-id>`, `--name <filename-pattern>`
- At least one filter required (no accidental approve-everything)

**Approve flow for each matched image:**
1. Read the staged PNG from staging/images/{subdir}/{filename}.png
2. Determine target path in public/images/:
   - service-hero, city-hero -> public/images/heroes/{filename}.webp
   - gallery-before, gallery-after -> public/images/gallery/{filename}.webp
   - content-* -> public/images/content/{filename}.webp
   - homepage -> public/images/homepage/{filename}.webp
3. Process through sharp pipeline: resize to target dimensions (from PromptDefinition), convert to WebP quality 80
4. Create public/images/{subdir}/ directory if it doesn't exist
5. Write optimized WebP to target path
6. Build ImageEntry object with: id, filename, path (relative to public/), alt, width, height, category, pageType, pageId, prompt (sceneDescription), generatedAt (ISO timestamp)
7. After all images processed, call addToManifest() with new entries

**Post-approve:**
- Delete approved images from staging/
- Print summary: X images approved, paths written, manifest updated
- If all staged images approved, delete staging/preview.html too

**Error handling:**
- If sharp processing fails for an image, log error and skip (don't abort batch)
- If image already exists in public/images/ at target path, overwrite it (regeneration case)

**scripts/images/status.ts** -- Status reporting:
- Count images in staging/images/ (staged, pending review)
- Count images in public/images/{heroes,gallery,content,homepage}/ (approved, in production)
- Read manifest and count entries by category
- Calculate coverage: service heroes (X/65), city heroes (X/21), homepage (X/15), gallery pairs (X/8), content pool (X/20)
- Show estimated total cost of all generated images (from reports in scripts/reports/)
- Print formatted table with category, staged, approved, total needed, coverage percentage
