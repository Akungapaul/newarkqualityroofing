Create the foundation type system and shared libraries for the image generation pipeline.

**scripts/images/lib/types.ts** -- Shared types:
- `ImageCategory` enum: 'homepage', 'service-hero', 'city-hero', 'gallery-before', 'gallery-after', 'content-crew', 'content-materials', 'content-consultation', 'content-seasonal', 'og'
- `PageType` enum: 'homepage', 'service', 'city', 'combo', 'shared'
- `ImageDimensions` type with presets: HERO (1920x1080), CONTENT (1200x900), GALLERY (1200x800), CARD (800x800), OG (1200x630), API_LANDSCAPE (1536x1024), API_SQUARE (1024x1024)
- `PromptDefinition` interface: id, filename, category, pageType, pageId (optional), targetWidth, targetHeight, sceneDescription, alt (SEO-optimized alt text)
- `ImageEntry` interface (matching Zod schema in manifest): id, filename, path, alt, width, height, category, pageType, pageId, prompt, generatedAt
- `GenerationResult` interface: id, status ('success' | 'failed' | 'skipped'), stagingPath, error (optional), costEstimate, durationMs
- `GenerationReport` interface: timestamp, totalRequested, succeeded, failed, skipped, estimatedCost, durationMs, results array

**src/data/image-manifest.ts** -- Central manifest (committed to git):
- Zod `ImageEntrySchema` validating all fields
- `imageManifest: ImageEntry[]` export (starts empty, populated by approve script)
- Lookup helpers: `getServiceHeroImage(serviceId)`, `getCityHeroImage(cityId)`, `getOGImage(pageType, pageId)`, `getHomepageImages()`, `getGalleryPairs()`, `getContentPoolImages(subcategory)`
- Each helper returns typed `ImageEntry | undefined` (or array for multi-image lookups)

**scripts/images/lib/openai-client.ts** -- OpenAI wrapper:
- Load OPENAI_API_KEY via dotenv
- `generateImage(prompt: string, size: '1536x1024' | '1024x1024')` function using openai SDK v6.x with `model: 'gpt-image-1'`, `quality: 'medium'`, `output_format: 'png'`, `n: 1`
- Returns `Buffer` (decoded from base64 response)
- Immediate disk write after decoding to prevent memory accumulation
- Exponential backoff on 429 errors (initial 5s, max 60s, 3 retries)
- Configurable concurrency via `generateWithConcurrency(items, concurrency=3, delayMs=1000)` using worker pool pattern from research
- `isRateLimitError(error)` helper

**scripts/images/lib/sharp-pipeline.ts** -- Image processing:
- `processImage(inputPath, outputPath, targetWidth, targetHeight)` -- resize with fit 'cover' + position 'centre', convert to WebP quality 80, effort 6
- `getImageInfo(path)` -- returns dimensions and file size for reporting
- Uses sharp library

**scripts/images/lib/cost-tracker.ts** -- Cost tracking:
- Cost constants: LANDSCAPE_MEDIUM = 0.05, SQUARE_MEDIUM = 0.034 (from research pricing)
- `estimateCost(prompts: PromptDefinition[])` -- calculates total estimated cost
- `generateReport(results: GenerationResult[], startTime)` -- creates GenerationReport object
- `writeReport(report, outputDir='scripts/reports')` -- writes JSON report to scripts/reports/generation-TIMESTAMP.json

**scripts/images/lib/manifest.ts** -- Manifest operations:
- `readManifest()` -- reads and parses src/data/image-manifest.ts
- `writeManifest(entries: ImageEntry[])` -- writes updated manifest to src/data/image-manifest.ts preserving the module structure (imports, schema, lookup helpers) and only updating the `imageManifest` array
- `addToManifest(newEntries: ImageEntry[])` -- reads existing, merges (by id, replacing duplicates), writes back
- Uses AST-free string replacement: find the `imageManifest` array content between markers, replace with serialized entries

Note on photo treatment: The style prefix uses "soft natural daylight with slightly warm undertones" (NOT "golden hour") to avoid double-warming with the CSS .photo-treatment overlay per research Pitfall 5.
