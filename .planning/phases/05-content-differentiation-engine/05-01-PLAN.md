Create `src/data/combo-content/schema.ts` with the ComboContentSchema using Zod:

```typescript
import { z } from 'zod';

export const ComboContentSchema = z.object({
  serviceId: z.string(),
  cityId: z.string(),
  overview: z.array(z.string()).min(3).max(5),
  challenges: z.array(z.string()).min(2).max(4),
  process: z.array(z.string()).min(2).max(4),
  faqs: z.array(z.object({
    question: z.string(),
    answer: z.string(),
  })).min(3).max(5),
  metaDescription: z.string().max(160),
});

export type ComboContent = z.infer<typeof ComboContentSchema>;
```

Create `scripts/validate-similarity.ts` as a standalone CLI script. Run via `npx tsx scripts/validate-similarity.ts`.

The script should:
1. Import all available combo content via getComboContent or by directly importing city index files
2. Group combos by serviceId and by cityId
3. For each group, compare all pairs using Dice-Sorensen coefficient via cmpstr
4. Apply tiered thresholds:
   - Same service across different cities: < 0.70 (70%)
   - Same city across different services: < 0.50 (50%)
   - Same category in same city: < 0.60 (60%)
5. Output a console table with columns: Page A, Page B, Type (cross-city/cross-service/cross-category), Score, Status (PASS/FAIL)
6. Print summary: total pairs checked, passes, failures, worst offenders

For rendering text content from a ComboContent object, concatenate all text fields:
```typescript
function renderText(c: ComboContent): string {
  return [
    ...c.overview,
    ...c.challenges,
    ...c.process,
    ...c.faqs.map(f => `${f.question} ${f.answer}`),
  ].join(' ');
}
```

Important: Check the actual cmpstr API. The research noted the exact method name needs verification. If neither works, implement a simple bigram Dice coefficient manually (the algorithm is straightforward: 2 * |intersection of bigrams| / |bigrams in A| + |bigrams in B|).
