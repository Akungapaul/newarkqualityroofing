  <files>scripts/validate-click-depth.ts, scripts/validate-flat-urls.ts, scripts/validate-topical-map.ts, scripts/audit-cannibalization.ts</files>
  <action>
1. Create `scripts/validate-flat-urls.ts`:
   - Import `getAllSlugs` from slug-registry
   - Check every slug for path separators (no `/` in any slug)
   - Also validate homepage (empty string is fine)
   - Exit 0 if all pass, exit 1 if any violations
   - Print summary: "Checked {N} slugs. {M} violations found."
   - Print violations if any

2. Create `scripts/validate-click-depth.ts`:
   - Build an adjacency graph from the DATA LAYER (not HTML parsing), following the established project pattern from scripts/audit-orphan-pages.ts
   - Link sources to include in the graph:
     a. **Header nav**: service menu groups (all services linked from dropdown), city menu items (all cities), comparison groups
     b. **Footer links**: all city links, major service category links from footer mega-links
     c. **Template internal links**: For each page type, compute its outbound links:
        - Service pages: related services (same category), Learn More article link, related comparisons
        - City pages: nearby cities, services grid (links to combo pages), city page internal links
        - Combo pages: nearby city combo links, related service combo links, parent page links (service + city)
        - Comparison pages: related comparisons
        - Article pages: money page link (parent), next article link, prev article link
        - Core pages: services hub links to all services, locations hub links to all cities, sitemap links to everything
     d. **Homepage links**: services grid links, locations grid links, comparison links, article links, browse services links
   - BFS from homepage ('/'), track depth for each node
   - Fail (exit 1) if any page has depth > 3
   - Print summary: "Total pages: {N}. Max depth: {D}. Pages at depth 1: {N1}, depth 2: {N2}, depth 3: {N3}."
   - If violations, print the offending pages with their depth
   - NOTE: Import from data modules and linking/link-engine. Build the graph programmatically. Do NOT fetch/parse HTML.

3. Create `scripts/validate-topical-map.ts`:
   - Import services and articles
   - For each service (pillar page), count supporting pages that link to it:
     - Articles with `parentType === 'service'` and `parentId === service.id`
   - Fail (exit 1) if any service has fewer than 3 supporting articles
   - Also verify comparison pages have articles:
     - Articles with `parentType === 'comparison'` and matching parentId
   - Print summary: "Pillar pages: {N}. All have 3+ supporting articles: {yes/no}."
   - Print any services with fewer than 3 articles

4. Create `scripts/audit-cannibalization.ts`:
   - Import all data: services, cities, combos, comparisons, articles
   - For each page, extract its "target keyword signal" from metaTitle + H1 pattern:
     - Service: service.name
     - City: "Roofing Services in {city.name}"
     - Combo: "{service.name} in {city.name}"
     - Comparison: comparison.metaTitle
     - Article: article.metaTitle
   - Compare every pair within the same page type and across related types using simple bigram similarity (reuse the Dice-Sorensen pattern from scripts/validate-similarity.ts)
   - Flag pairs with > 70% title similarity as potential cannibalization
   - Group results by severity:
     - HIGH: same page type, > 80% similarity
     - MEDIUM: same page type, 70-80% similarity
     - LOW: cross-type, > 70% similarity
   - Output: total pairs checked, high/medium/low flags, specific pairs
   - This is a REPORT script (does not fail build) -- informational only
   - NOTE: Cross-type cannibalization between combo and service pages is EXPECTED (service "Roof Repair" vs combo "Roof Repair in Newark, NJ" will be similar but serve different intents). The script should note this pattern and not count them as HIGH severity.

All scripts should use `npx tsx scripts/{name}.ts` to run. Follow the established pattern from scripts/audit-orphan-pages.ts for module imports and console output formatting.
