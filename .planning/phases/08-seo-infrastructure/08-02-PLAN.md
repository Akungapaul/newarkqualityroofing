1. Create `src/components/seo/JsonLd.tsx`:
   - Server component (no 'use client')
   - Props: `{ data: Record<string, unknown> }`
   - Renders `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(data) }} />`

2. Create `src/lib/schema.ts` with these builder functions (all using `siteConfig` and `SEO_CONFIG`):

   a. `buildPostalAddress()` -- internal helper returning PostalAddress from siteConfig.address

   b. `buildOrganizationSchema()` -- @type Organization, @id `${BASE_URL}/#organization`, name, url, telephone, email, address

   c. `buildRoofingContractorSchema()` -- @type RoofingContractor, @id `${BASE_URL}/#roofingcontractor`, name, url, telephone, email, priceRange '$$', address, aggregateRating (ratingValue '5.0', reviewCount '500'), openingHoursSpecification (Mon-Fri 7-18, Sat 8-14)

   d. `buildWebSiteSchema()` -- @type WebSite, @id `${BASE_URL}/#website`, url, name, publisher @id ref to organization

   e. `buildWebPageSchema(url: string, name: string)` -- @type WebPage, @id `${url}/#webpage`, url, name, isPartOf @id ref to website

   f. `buildLocalBusinessSchema(city: { name: string; state: string; zipCodes: string[] })` -- @type RoofingContractor (subtype of LocalBusiness), @id `${BASE_URL}/#roofingcontractor`, name, url, telephone, address (HQ), areaServed with @type City, city.name, containedInPlace @type State NJ. Include priceRange, aggregateRating, openingHours same as buildRoofingContractorSchema.

   g. `buildServiceSchema(service: { name: string; slug: string; shortDescription: string })` -- @type Service, @id `${BASE_URL}/${service.slug}/#service`, name, description, provider @id ref to organization, areaServed AdministrativeArea 'Essex County, NJ'

   h. `buildFaqSchema(faqs: Array<{ question: string; answer: string }>)` -- @type FAQPage, mainEntity array of Question with acceptedAnswer Answer

   i. `buildBreadcrumbSchema(items: Array<{ name: string; url?: string }>)` -- @type BreadcrumbList, itemListElement with position, name, and item (url) only when url is present (last item has no url per schema.org spec)

   j. `buildArticleSchema(article: { title: string; slug: string; description: string })` -- @type Article, @id `${BASE_URL}/${article.slug}/#article`, headline, description, author @id ref to organization, publisher @id ref to organization

   k. `buildJsonLdGraph(...schemas: Record<string, unknown>[])` -- wraps all schemas in `{ '@context': 'https://schema.org', '@graph': schemas }`

   All @id references must be consistent so Google can connect entities across the @graph.
   Do NOT add schema to thank-you or privacy-policy pages (they are noindex).
