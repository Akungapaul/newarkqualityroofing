- Approach: Wrap the content `<div>` (left side) in a motion.div with parent variants:
  ```
  heroVariants = { hidden: { opacity: 0 }, visible: { opacity: 1, transition: { staggerChildren: 0.2, delayChildren: 0.1 } } }
  itemVariants = { hidden: { opacity: 0, y: 20 }, visible: { opacity: 1, y: 0, transition: { duration: 0.5, ease: 'easeOut' } } }
  ```
- Since HeroSection needs to stay server, use the AnimateIn wrapper on each child with delay props:
  - Image badge: delay=0.2
  - h1 heading: delay=0.4
  - Subtitle paragraph: delay=0.6
  - Phone CTA: delay=0.8
  - Lead form container: delay=0.4 (slides in from right — use a separate AnimateIn-like wrapper with initial={{ opacity: 0, x: 30 }})
- For the form slide-from-right effect, create a small inline client component `HeroFormReveal` that wraps the form div with `initial={{ opacity: 0, x: 30 }}, whileInView={{ opacity: 1, x: 0 }}, viewport={{ once: true }}, transition={{ duration: 0.6, delay: 0.4, ease: 'easeOut' }}`
- CRITICAL: No layout shift. All containers must have their dimensions pre-reserved. The hero already has min-h-[85vh], so CLS is not a concern.

**2. Header.tsx — scroll-based shrink/shadow:**
- Header is already 'use client'. Add scroll detection:
- Add state: `const [scrolled, setScrolled] = useState(false);`
- Add useEffect with passive scroll listener:
  ```
  useEffect(() => {
    const handler = () => setScrolled(window.scrollY > 50);
    window.addEventListener('scroll', handler, { passive: true });
    return () => window.removeEventListener('scroll', handler);
  }, []);
  ```
- Update header element classes dynamically:
  - Default (not scrolled): py-3 (current)
  - Scrolled: py-1.5, shadow-lg, backdrop-blur-md, bg-forest/95
  - Add `transition-all duration-300` to the header element for smooth interpolation
- The header already has `sticky top-0 z-40 bg-forest shadow-lg`. Modify:
  - Remove the static `shadow-lg` and `border-b`
  - Apply conditionally: `${scrolled ? 'py-1.5 shadow-lg shadow-forest-dark/30 backdrop-blur-md bg-forest/95 border-b border-forest-dark/20' : 'py-3 bg-forest'}`
  - The transition between states should feel smooth (CSS transition handles it)

**3. FloatingCtaButton.tsx — scroll-triggered entrance:**
- Currently shows/hides based on viewport size only (lg:hidden). Add scroll-triggered visibility:
- Import { useScroll, useTransform, motion } from 'motion/react'
- Track scrollYProgress and show the button when user scrolls past approximately 100vh (hero fold):
  ```
  const { scrollY } = useScroll();
  const [pastFold, setPastFold] = useState(false);
  useEffect(() => {
    return scrollY.on('change', (v) => setPastFold(v > 600));
  }, [scrollY]);
  ```
- Wrap the button container in motion.div:
  ```
  initial={{ y: '100%' }}
  animate={pastFold ? { y: 0 } : { y: '100%' }}
  transition={{ duration: 0.3, ease: 'easeOut' }}
  ```
- Remove the static visibility — it should be hidden until scroll past fold, then slide up
- Keep lg:hidden for desktop (on desktop, use a fixed bottom-right pill instead):
  - Mobile: full-width bottom bar (existing design)
  - Desktop: `fixed bottom-6 right-6 rounded-full px-6 py-3 shadow-xl` pill button, also with slide-up entrance
- The button should also hide when user scrolls back to top (pastFold becomes false)
