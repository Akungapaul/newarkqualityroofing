<task type="auto">
  <name>Task 2: Create animation wrapper components and wire MotionConfig</name>
  <files>src/components/animations/AnimateIn.tsx, src/components/animations/CountUp.tsx, src/components/animations/StaggerGrid.tsx, src/app/layout.tsx</files>
  <action>
1. Create `src/components/animations/AnimateIn.tsx`:
- 'use client' directive
- Import { motion } from 'motion/react'
- Props: children (ReactNode), className (optional string), delay (optional number, default 0), as (optional keyof JSX.IntrinsicElements, default 'div')
- Uses motion.div with: initial={{ opacity: 0, y: 20 }}, whileInView={{ opacity: 1, y: 0 }}, viewport={{ once: true, amount: 0.2 }}, transition={{ duration: 0.6, ease: 'easeOut', delay }}
- Passes className through to motion.div
- Export named: AnimateIn

2. Create `src/components/animations/CountUp.tsx`:
- 'use client' directive
- Import { useInView } from 'motion/react'
- Props: target (number), suffix (string, default ''), prefix (string, default ''), duration (number, default 1500)
- Uses useRef + useInView(ref, { once: true }) to detect viewport entry
- On viewport entry, uses requestAnimationFrame loop with easeOut cubic: `1 - Math.pow(1 - progress, 3)`
- Renders: <span ref={ref}>{prefix}{value}{suffix}</span>
- Handles prefers-reduced-motion: if reduced motion, show final value immediately (no animation)
- Export named: CountUp

3. Create `src/components/animations/StaggerGrid.tsx`:
- 'use client' directive
- Import { motion } from 'motion/react'
- Two exports:
  - StaggerGrid: motion.div with variants using staggerChildren: 0.1, delayChildren: 0.2. Props: children, className, as (default 'div'). Uses initial="hidden" whileInView="visible" viewport={{ once: true }}.
  - StaggerItem: motion.div with item variants: hidden={{ opacity: 0, scale: 0.97 }}, visible={{ opacity: 1, scale: 1, transition: { duration: 0.4, ease: 'easeOut' } }}. Props: children, className.
- Container variants: hidden={{ opacity: 0 }}, visible={{ opacity: 1, transition: { staggerChildren: 0.1, delayChildren: 0.2 } }}

4. Update `src/app/layout.tsx`:
- Import { MotionConfig } from 'motion/react' — this requires 'use client' on the component that renders MotionConfig. Since layout.tsx should remain a server component, create a small client wrapper:
  - Option A: Create `src/components/animations/MotionProvider.tsx` ('use client', wraps children in `<MotionConfig reducedMotion="user">{children}</MotionConfig>`)
  - Then import MotionProvider into layout.tsx and wrap {children} with it
- This ensures ALL motion components on the site automatically respect prefers-reduced-motion without per-component handling.

IMPORTANT: Do NOT make layout.tsx a client component. The MotionProvider wrapper keeps the client boundary minimal.
